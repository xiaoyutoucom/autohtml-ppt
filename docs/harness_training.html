02

认识 Harness

先讲清是什么，再拆五大模块。

Why Harness

用 AI 编程，这些坑你踩过吗？

01 · 改偏

只想改样式

AI 重写整个布局 —— 行为跑偏。

02 · 忘约

约定 ≤200 行

聊十几轮后写成 1000 行 —— 约束蒸发。

03 · 越修越乱

修 1 个 Bug

又来 3 个新 Bug，代码越改越乱。

行为层三坑：改偏、忘约、越修越乱。下一页还有更隐蔽的：幻读、记忆篡改 —— 单靠提示词救不了。
Why Harness · 续

Skill 写了也挡不住？更隐蔽的坑

幻读 / 幻觉

Skill 写了，照样「幻读」

规则 / Skill 明明写着路径与约束，AI 仍声称已读文件、编造 API、瞎编目录 —— 文档约束没有强制执行力。

记忆污染

旧记忆篡改当前需求

根据历史对话「以为你还要 X」，擅自改接口 / 改命名 / 改流程；跨会话记忆一串味，就把新任务带偏。

假装执行

口头合规，实际没跑

说「已按 Skill 做完 / 已自测通过」，但没真读、没真跑、没真验 —— 没有门禁就无法证伪。

约束失效

只写在文档里 = 可被忽略

AGENTS.md / Skill 是「建议」不是「铁律」。没有 Lint、测试、阶段门禁，约束会在长对话里逐渐蒸发。

解法不是再写一篇更长的 Skill —— 而是用 Harness：落盘状态 · 可验证门禁 · 按阶段注入上下文 · 阻断错误推进。
Definition

Harness = 马具

模型是马；Harness 是缰绳、路线、围栏 —— 让它跑得更快、更稳。

  • 规则文件 · 工具配置 · 任务拆分顺序 · 测试检查流程
  • 统统算 Harness：围绕模型的工作环境 + 工作流程
  • ①上下文 ②执行 ③编排 ④反馈 ⑤护栏 —— 五个核心模块
目标：让这匹马顺利完成任务，而不是在野地里乱跑。

03

五大核心模块

并不抽象 —— 就是 AI 干活时会遇到的几个核心问题；有项目经验的人会觉得很熟。

模块 ①

上下文架构：了解背景和规矩

像新人入职：先喂需求、背景、开发规范。

Before · 坑

几千行塞一个文件

  • 规则全堆进一个 AGENTS.md
  • AI 更容易漏关键信息
  • 长对话里约束逐渐蒸发
After · 改法

AGENTS.md 当目录

  • 写清技术栈、规范、禁止事项
  • 细节放 docs/，按需加载
  • 分层文档 · 上下文压缩
关键词：AGENTS.md · 分层文档 · 上下文压缩 · 渐进式加载
模块 ②

执行能力：装上手脚和工具

模型本身只会输出文本 —— 要开发项目，得能操作电脑。

CLI

终端

跑命令、装依赖、看日志。

FS

文件

读写代码与配置。

WEB

浏览器

实机打开、点选、验收。

MCP

MCP

联网、读库、抓最新内容。

SKILL

Skills

复杂工作流封成技能包。

工具越多，能干的活越多 —— 同时要有权限边界。
模块 ③

任务编排：安排好工作计划

一把梭大需求 → 上下文装不下、约束被冲淡 → 渣渣代码。

01

拆小任务

每次只做一个功能点

02

Plan Mode

出方案，人工确认再写码

03

沉淀 + Git

每功能存档，可回滚

04

SubAgents

互不依赖时并行

一个一个来，稳步推进 —— 新对话靠文档找回记忆。
模块 ④

反馈机制：自己检查自己的工作

AI 说「完成了」≠ 一点运行就全是 Bug。

环节做什么谁介入
自动检查Linter / 测试 / 浏览器Harness
自主修复读报错 → 改代码Agent
搞不定描述 Bug 求助人
互审多 Agent ≈ Code ReviewAgent × N
Harness 不是放手不管 —— 把人的精力用在最关键的地方。
模块 ⑤

架构护栏：防止越改越乱

AI 会模仿仓库里已有代码 —— 包括烂代码；技术债越滚越大。

01

架构 Linter

限制依赖方向（≠ 风格 Linter）

02

Pre-commit

提交前自动拦截违规

03

垃圾回收

定期扫描并提修复 PR

04

Git 检查点

每功能提交，随时可回滚

Summary

Harness = 工程经验，系统化用到 AI 上

不是新技术

写规则、配工具、拆任务、跑测试、定架构 —— 程序员做项目的老方法；换到 AI 场景，就叫 Harness。

越懂工程，越能驾驭 AI

多做完整项目、多积累工程经验 —— 需求分析、方案设计、任务拆解、质量把关，就是最好的 Harness。

开工前写好 AGENTS.md
先 Plan Mode，人工确认再写码
MCP / Skills 配好工具
做完功能自己跑测试 / Linter / 浏览器
每功能：沉淀文档 + Git 存档点
